## Installation

Install via `uv` (recommended):

```bash
uv tool install kscli
```

Or via `pip`:

```bash
pip install kscli
```

**Requirements:** Python 3.12+

## Quick start

### 1. Get an API key

Create an API key from the Knowledge Stack web app: **My Account > API Keys**.

### 2. Authenticate

```bash
kscli login --api-key sk-user-your-key-here
```

Or run `kscli login` and paste the key when prompted.

### 3. Verify

```bash
kscli whoami
```

### 4. Start using it

```bash
# List your folders
kscli folders list

# Upload a document
kscli documents ingest --file report.pdf --path-part-id <folder-path-part-id>

# Search your knowledge base
kscli chunks search --query "quarterly revenue analysis"

# View document structure
kscli document-versions contents <version-id> --format tree
```

## Key features

- **Resource-first commands** — every resource (`folders`, `documents`, `chunks`, etc.) has consistent verbs: `list`, `describe`, `create`, `update`, `delete`
- **Multiple output formats** — table (default), JSON, YAML, tree view, or ID-only (for scripting)
- **Bulk operations** — upload entire directory trees with `kscli folders bulk-ingest`
- **Semantic search** — search by meaning with `kscli chunks search`
- **Workflow monitoring** — track document processing with `kscli workflows list`
- **Scriptable** — use `--format json` or `--format id-only` for piping into other tools

## Output formats

Control output with `--format` (or `-f`):

| Format | Description | Use case |
|--------|-------------|----------|
| `table` | Rich formatted table | Interactive use (default) |
| `json` | Pretty-printed JSON | Scripting, API debugging |
| `yaml` | YAML output | Configuration, readability |
| `id-only` | One ID per line | Piping into other commands |
| `tree` | Hierarchical tree view | Viewing folder/document structure |

Suppress table headers with `--no-header` (or `-H`).

## Examples

### Browse and search

```bash
# List root folders
kscli folders list

# List contents of a specific folder (folders + documents)
kscli folders list --folder-id <id> --show-content

# View folder as a tree
kscli folders list --folder-id <id> --show-content --format tree

# Search by name
kscli documents list --parent-path-part-id <folder-id>

# Semantic search across all content
kscli chunks search --query "how does authentication work"

# Keyword search scoped to a folder
kscli chunks search --query "ERROR-4021" --search-type full_text --parent-path-ids <folder-id>
```

### Upload and process

```bash
# Upload a single document
kscli documents ingest --file presentation.pptx --path-part-id <folder-path-part-id>

# Bulk upload a directory
kscli folders bulk-ingest ./reports --folder-id <folder-id> --extensions .pdf,.docx

# Preview a bulk upload (dry run)
kscli folders bulk-ingest ./reports --folder-id <folder-id> --dry-run

# Monitor processing
kscli workflows list
kscli workflows describe <workflow-id>
```

### Manage content

```bash
# Create a folder
kscli folders create --name "Q4 Reports" --parent-path-part-id <parent-id>

# View document versions
kscli document-versions list --document-id <doc-id>

# View document structure (sections and chunks)
kscli document-versions contents <version-id> --format tree

# View with inline content
kscli document-versions contents <version-id> --show-content

# Tag a document
kscli tags create --name "Important" --color FF0000
kscli tags attach <tag-id> --path-part-id <document-path-part-id>
```

### Conversations

```bash
# Create a thread
kscli threads create --title "Research discussion"

# List your threads
kscli threads list

# Add a message
kscli thread-messages create --thread-id <id> --content "What are the key findings?" --role USER
```

### Administration

```bash
# List tenant members
kscli tenants list-users <tenant-id>

# Invite a user
kscli invites create --email user@example.com --role USER

# Grant folder access
kscli permissions create --user-id <uid> --path-part-id <folder-id> --capability READ_WRITE

# Cancel a stuck workflow
kscli workflows cancel <workflow-id>
```
